Webserv Parsing Guide

Step by step technical breakdown (｢• ω •)｢

The parsing process transforms the .config file into C++ structures.
It identifies server blocks, ports, and specific route configurations.
This system ensures that each request reaches the correct virtual server.

Parsing Workflow

  • Reader: Opens the file and generates tokens (words and symbols).
  • Config: The main class that groups servers by their port.
  • ServerConfig: Stores data for a single server {} block.
  • Location: Stores rules for specific URL paths and methods.

Key Class Attributes

  • _servers: Map that handles multiple ports (Config class).
  • _error_pages: Map for custom error HTMLs (ServerConfig class).
  • _methods: List of allowed GET/POST/DELETE (Location class).
  • _cgi_info: Map for script executors (Location class).

The Default Server Rule

The first server defined for a port is stored at index [0].
If no server_name matches, index [0] handles the request.
