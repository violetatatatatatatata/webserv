Webserv Error Handling

Exception-based error management (｢• ω •)｢

Our server uses C++ exceptions to handle HTTP error states.
This allows us to stop execution and jump directly to the response phase.
Custom HTML pages are then served based on the specific error code.

1. The Exception Hierarchy

  • HttpException: The base class that stores the HTTP status code.
  • Client Errors (4xx): Exceptions like 404 (Not Found) or 403 (Forbidden).
  • Server Errors (5xx): Exceptions like 500 (Internal) or 504 (Gateway Timeout).

2. Implementation Flow

  • Throw: When a file is missing or a CGI fails, we throw an exception.
  • Catch: The main loop catches HttpException to prevent crashes.
  • Response: The Response class builds the header and loads the HTML.

3. Custom Error Pages

The server looks for custom pages in the error_page directive.
If not found, it falls back to our default files in www/client_error/.
